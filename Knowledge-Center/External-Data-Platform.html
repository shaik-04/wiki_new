External Data Platform

Atom.AI

About

External Data Platform is a platform where we can add and access data from outside sources. An External Data Platform helps you to discover and use thousands of relevant data signals which will improve predictions and machine learning model performance.

Steps to add and view External Data in Atom

  1. Exploring External Data Platform in Atom.Ai
  2. Adding new External Data to Atom.Ai
  3. Preview the External Data and data cataloguing
  4. Analyzing the Data through Data Bricks

 

Exploring External Data Platform in Atom.Ai

     Navigation – Atom.AI

     Link – Click here

A new window will pop-up once you click on the external data platform icon from the Atom.Ai homepage.

This window will tell the complete details of all the external data that are present in Atom.Ai. 

  • The first column “Data Provider” will show the Data Provider source or organization from where the data is taken from. 
  • Next column, “Requested Date” is the date at which the data is added to the Atom.Ai, 
  • “Security Agreement” is the agreement between the third party from where the data is being collected and Atom.Ai, where data is adding. This agreement will contain all the terms and conditions between both the parties. 
  • “Status” will show the approval status of the external data. 
  • The last column “Action” will allow user to download the security agreement. 

 

Adding new External Data to Atom.Ai

     Navigation – Atom.AI ⇢ External Data Platform

     Link – Click here

In order to add new external data click on the Add Provider tab

A pop-up window will appear to fill out the new provider details. In order to create new provider, the user needs to give Provider name, it's description, the contract end date after which it can be renewed according to the Payment Frequency, the cost of the data provider and Payment Frequency which can be monthly or yearly. User also needs to upload the security agreement file and click on save to create the new data provider.

After adding any new provider to the External Data Platform, the request will go to the admin for approval and till that time the status will be pending. Once the admin will approve the request, the status will be changed from pending to approved and a notification will be received to the request creator along with the admin notes where the admin will mention its email id to the user for getting the link of the data for addition in the Atom.Ai portal.

User can check the uploaded data by clicking on the “Storage Explorer” tab.

After clicking on the Storage Explorer tab it will take you to the login page where after login, it will take you to the Microsoft Azure storage container page.

Click on the Landing folder option.

Now click on the Restricted option

You will be able to see your created Data Provider in this window. For Example, you have created “Decadata” as a Data Provider or you want to see the data in Decadata. Click on the Decadata folder.

Now, click on decadata_raw_history.

Here, you can see all the data that are present under the Decadata along with its modification time, Blob type, size and Access tier. You can also upload new data by clicking on the upload tab and also can add directory in this folder.

 

Preview the External Data and data cataloguing

     Navigation – Atom.AI ⇢ External Data Platform

     Link – Click here

By clicking on the “Data Catalog” tab and after logging in the subsequent window, Microsoft Purview page will be opened. This page will helps you in browsing and searching your data assets. The data catalog aligns your assets with friendly business terms and data classification to identify data sources.

The Microsoft Purview Data Catalog provides data curation features such as business glossary management and the ability to automate tagging of data assets with glossary terms. Data consumers and producers can also visually trace the lineage of data assets: for example, starting from operational systems on-premises, through movement, transformation & enrichment with various data storage and processing systems in the cloud, to consumption in an analytics system like Power BI.

In this page you will have three tabs, “Browse assets”, “View glossary” and “Knowledge center”. In the “View glossary” option you can view the details of the team's business glossary. In the “Knowledge center”, there are many videos, blogs and tutorials which will help you in getting familiar with the Microsoft Purview.

“Browse assets” is the tab where you can explore your assets. click on the Browse assets tab or alternatively, you can also click on the browse option present in the left panel.

Now, click on the External-Data-Platform.

A new window will open showing all the external data's. You can click on the required data source. For the reference, click on the Decadata Source.

In this window, you will get all the details of the Decadata source. In the Overview tab, you will get Asset description, it's managed attributes, Classifications, Schema classifications and Fully qualified name

In the properties section, details like property description of the data along with a filter on property key is present.

The Schema tab will show all the schemas associated with the data source. This will include the schemas name, it's data type, it's Classifications, sensitivity label, etc.

This window will tell about the Lineage that are present with the data source.

This window will give you the contact details of the people whom you can contact about the data assets.

This window will give all the related information which are associated with the data assets.

 

Analyzing the Data through Data Bricks

     Navigation – Atom.AI ⇢ External Data Platform

     Link – Click here

Click on the “Data Analysis” tab to open the Azure Databricks window. After validating the credentials, the following window will open.

In this window, you can create your own workspace by clicking on the workspace option from the left panel. You can execute your queries here and can do the analysis of the external data's.
