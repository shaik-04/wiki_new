Databricks Feature Store Deployment

Atom.AI

 

Databricks feature store pipeline allow user to register Databricks feature store based on the selected solution in the Atom.ai canvas.

Deploy azure Databricks workspace

Select premium tier and after successful deployment go to the workspace and create a cluster. During cluster creation select the configuration shown in the image.

Use 11.1 ML (includes Apache Spark 3.3.0, Scala 2.12) as  Databricks Runtime Version, min workers as 1, max workers as 2, and cluster time out as 15 min.

After successful workspace and cluster creation. Please note down the following values: 

  1. workspace URL 
  2. cluster ID 
  3. workspace ID.

 

For making it feature store  the following pipeline should be run with the variables changes

Pipeline URL : https://dev.azure.com/Studio-Org/Deployment-Studio/_build?definitionId=751

Repo URL: https://dev.azure.com/Studio-Org/Deployment-Studio/_git/infra_test_dbx-fs-pipeline?version=GBmain&path=/README.md&_a=preview

Variables that should be changed

DBX specific pipeline variables

$(cluster-id)             :                    Databricks cluster id 

$(databricks-host)       :               Databricks host endpoint

$(databricks-token)     :               Databricks PAT token

$(databricks-workspace-id) :     Databricks workspace ID

Choose a storage account to dump your features and add them as variables in pipeline

ADLS specific pipeline variables

$(container-name)       :               ADLS container name

$(storage-account-name) :         ADLS storage account name

$(storage-key)          :                    ADLS storage account secret key

Feature store specific pipeline parameters (not required to change)

$(fs-scope):                                   Feature store scope

$(fs-key):                                        Feature store key
