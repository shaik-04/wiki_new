How to Clone?

Atom.AI

 USER ACTION REQUIRED

 

Selecting the desired schemas to the cart

  1. Open homepage of Atom.Ai
  2. Select any one collection of db schemas which you wanted to add to the cart
    1. System will load it into the next window – Showing the list of schemas with the category they belongs     under different headers
  3. Selecting the desired schemas and adding it to the cart

Cloning the schemas

  1. Clicking on cart option to re-check the desired schemas
  2. Click on cloning to clone the schemas
    1. System will load it into the next window – Showing two options (Add to existing solutions or create new product)
    2. Choose the desired option

Choosing or creating new solutions

  1. If chosen ‘Existing Solution’
    1. Choosing UX works
    2. Choosing Algo works
    3. Choosing Data works
  2. If chosen ‘Create new Solution’
    1. System will load it into the next window – Fill the form, entering the solution details and click on submit
    2. Verifying the solution at backend

 

Selecting the desired schemas to the cart (user action)

     Navigation – Atom.AI

     Link – Click here

Click on any one of the schemas group whose schemas you wanted to add to the cart.

      Navigation – Atom.AI ⇢ solution  

      Link – Click here

Select the desired schema to the cart by clicking on the encircled basket option.

 

Cloning the schemas (user action)

      Navigation – Atom.AI ⇢ solution

      Link – Click here

Click on the cart button to re-check the selected schemas.

If you want to delete the unwanted added schemas from the cart, click on the delete icon next to the schemas and click on ok button.

Click on clone button to clone the selected schemas from the cart.

      Navigation – Atom.AI ⇢ solution ⇢ provision-overview

      Link – click here

New window to select the solution will pop-up.

  1. If you haven’t selected any schemas from the solutions group in your cart, then the first option to add to existing solutions will be enabled for you. In which you can search for existing solutions and select from it or you can also create new product.
  2. If you have any schemas from the solutions group in your cart, then the first option to add to existing solutions will be disabled for you and you have to compulsorily click to create new product.

 

Choosing existing Solutions

If you choose existing solutions then to verify that your schemas has been added to the backend, you can individually check at the backend for UX works, Algo works and Data works separately by following the steps.

Choosing UX works to create solutions(user action)  

      Navigation – Atom.AI ⇢ Discover Atom ⇢ UX works of choice ⇢ clone

      Link – Click here

Choose UX of your choice from the existing UX works and add them to cart

After expanding the cart icon on the right side, click on the ‘Clone’ button 

Click either on ‘oct Realse’ or ‘octClone’ or any other solution according to your choice present in the existing solutions list. It will redirect to the ‘My Solutions’ page. 

Choose the solution of your choice. Here, I chose ‘oct Realse’. 

Click on the ‘Solution Board’. It will redirect you to Azure DevOps page. 

On Azure DevOps page, choose ‘Branches’ from ‘Repos’. 

Choose ‘project’ for UX works. 

Select ‘All’ to see the new branch created for your solution.

Choose the solution of your choice. Here, I chose ‘oct Realse’.

Click on the ‘Solution Board’. It will redirect you to Azure DevOps page.

Creating new solutions(user action)  

      Navigation – Atom.AI ⇢ solution ⇢ provision-overview ⇢ new-solution

      Link – click here

Fill up the Solution details (to create new solution) : user action required.

a. Fill the desired solution name

b. Fill the desired solution short name

c. Select the industry from the drop-down menu

d. Select the solution type from the drop-down menu

e. Write other specific detail you want to add regarding the solution

f. Write a description about the solution

g. Select solution start date and end date from the calendar

h. Fill the budget of the solution

After filling all the details, click on submit button to create new solution.

 

Verifying the Solution at backend

Once, the solution is created, It will redirect to the ‘My Solutions’ page.

Navigation – Atom.AI ⇢ solution

Link – click here

Click on the newly created solution.

In the new window of solution overview, click on the Solution Board.

Azure DevOps window will open, click on the Repos tab from the left panel.

You can search for the added solution file from the top drop down menu. As shown in the image above, you can see the fork files which is actually the files that are associated with the solutions created. These files will have files of all Algo Works, Data Works and UX Works in the respective fork folder.

A fork is a copy of a repository. The main advantage of the Forking Workflow is that contributions can be integrated without the need for everybody to push to a single central repository. It means that if there are any changes made in the Atom.Ai schemas then it will automatically be reflected in the chosen solution i.e., the solutions are automatically synced with the original schemas.
